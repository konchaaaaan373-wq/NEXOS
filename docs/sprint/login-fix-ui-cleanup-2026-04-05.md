# Sprint: ログイン修正 + UI整理

## 目標
1. ログイン機能を正常に動作させる
2. ダッシュボードUIを整理して見やすくする

## タスク

### Task 1: ログイン機能修正
- login/page.tsx のメール/パスワードフォームで NextAuth signIn() を呼ぶ
- デモモード（DB無し）はsessionStorage方式を維持
- middleware のDB有無判定と認証フローの整合性を確保

### Task 2: ダッシュボードページ整理
- 冗長なセクション（空席インパクト + 今日のアクション）を統合
- ビューチャート → Analytics ページへ移動（ダッシュボードから削除）
- 最近の候補者 → 簡潔な3件表示 + 「もっと見る」リンク
- メトリクスカードを4つに絞る

### Task 3: 管理ページ整理
- タブUIで情報を分割（概要/ユーザー/アクティビティ/システム）
- クリニックカードのミニパイプラインを削除しシンプル化
- スクロール可能ユーザーリストを整理

### Task 4: サイドバー整理
- ナビゲーション項目をグループ分け
- 視覚的な区切りを追加

### Task 5: 全体デザイン統一
- 情報の階層を明確化（アラート > メトリクス > リスト）
- カード密度を下げ、余白を適切に確保
